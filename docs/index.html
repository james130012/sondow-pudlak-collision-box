Lean-checked conditional theorem

A proof-complexity collision framework for Euler's constant.

This project does not claim an unconditional proof of the irrationality of the Euler-Mascheroni constant. It formalizes the interface where a Sondow short-proof collapse and a Pudlak finite-consistency lower bound must meet on exactly the same proof-length coordinate.

γ
Scientific boundary. The Lean chain proves an implication: if the stated Sondow, Pudlak, encoding, projection, and proof-length calibration certificates are supplied, then the collision derives ¬ is_rational(γ).

The collision must happen on one object.

The key audit point is not just having an upper bound and a lower bound. Both inequalities must contain the same middle term.

Core inequality chain

Once the calibrations identify the middle term, the two sides combine into a single chain and therefore force the contradictory comparison.

L(n) ≤ LenPA(Cn) ≤ U(n) ⇒ L(n) ≤ U(n), but U(n) < L(n)

Certificate architecture.

The formalization keeps each difficult mathematical input visible instead of hiding it in informal prose.

Sondow side

A rationality-to-short-certificate mechanism: under γ ∈ ℚ, the Sondow criterion is organized as a short verification route.

Pudlak side

A finite-consistency or reflection lower-bound package supplying the long-proof obstruction for the target family.

Common box

Formula code, PA proof system, and symbol-size length measure are fixed so the upper and lower bounds are comparable.

What reviewers should ask.

The project is designed around explicit audit questions. These are the places where future internalization or literature matching matters.

  • Does Pudlak's Theorem 5 cover the exact formula family used by the local collision box?
  • Does the rescaling preserve a lower bound strong enough to beat the Sondow upper bound?
  • Is the abstract proof_length pointwise equal to the checked-code minimum size?
  • Do the payload-truth certificates express the intended finite-consistency or reflection content?
